理解定位

例1. 家谱

定位成功后，我们需要对定位得到的PyQuery对象进行操作

选择器(定位)表达式

Selector_Expression 选择器表达式例子 解释
class .welcome 定位<div class="welcome">的标签
id #python 定位<div id="python">的标签
tag meta 定位所有<meta charset="UTF-8">标签的
tag1 tag2 tag3 div div 定位<div id="wechat">内的两个子div标签
tag & attr div[class] 定位带有class属性div标签
tag & attr & attr_value div[id=wechat] 定位id=wechat的div标签

1. class

html标签中class值可以用来定位，定位方法

.classvalue

   

定位class值为welcome的div

doc('.welcome')

[<div.welcome>]

str(doc('.welcome'))

'<div class="welcome">\n        <p>欢迎各位同学学习《python爬虫与文本分析》这门课程</p>\n    </div>\n\n    '

2. id

html标签中id值也可以用来定位，定位方法

#python

定位id值为python的div

doc('#python')

[<div#python>]

str(doc('#python'))

'<div id="python">\n        <h1>python大法好</h1>\n    </div>\n    '

3. tag

html标签中标签名也可以用来定位，

  • 优点：定位条件宽泛
  • 缺点： 定位精确度较差

定位meta标签

doc('meta')

[<meta>]

str(doc('meta'))

'<meta charset="UTF-8"/>\n\n    '

4. 多层tag

由于使用单一的标签名定位，定位的唯一性较差，我们需要用一些技巧提高表达式的精确度。

定位div div标签

doc('div div')

[<div>, <div>]

str(doc('div div'))

'<div title="chat">公众号：大邓和他的Python</div>\n        <div title="imgs"><img src="a.jpg"/></div>\n    '

5. tag & attr

除了多层标签名方式提高表达式的精确度，还可以使用标签属性值来限定标签，提高表达式的精确度。

定位含有class的div标签

doc('div[class]')

[<div.welcome>]

str(doc('div[class]'))

'<div class="welcome">\n        <p>欢迎各位同学学习《python爬虫与文本分析》这门课程</p>\n    </div>\n\n    '

6. tag & attr & attr_value

我们还可以比5更加严格的条件来提高标签表达式的精确度。

定位id=wechat的div标签

doc('div[id=wechat]')

[<div#wechat>]

str(doc('div[id=wechat]'))

'<div id="wechat">\n        <div title="chat">公众号：大邓和他的Python</div>\n        <div title="imgs"><img src="a.jpg"/></div>\n    </div>\n\n    '

PyQuery常用方法

定位成功后，我们需要对定位得到的PyQuery对象进行操作

PyQuery对象方法 功能
PyQuery(Selector_Expression) 查找符合Selector_Expression条件的标签，返回PyQuery
PyQuery.items(Selector_Expression) 查找符合Selector_Expression条件的标签, 返回PyQuery列表
PyQuery.eq(index) 获得第 index+1 个标签
PyQuery.text() 获得标签内的文本
PyQuery.attr(attribute) 获得标签的属性值

2. PyQuery.items(Selector_Expression)

PyQuery.items(Selector_Expression)与PyQuery(Selector_Expression)区别

返回的结果for循环迭代时是否依旧是PyQuery类型
